### 토큰 저장 방식의 보안적 재설계

#### 인증 방식 비교

| 인증 모델 | 저장 위치 | 서버 상태 | 특징 | 단점 / 보안 문제 |
| --- | --- | --- | --- | --- |
| **서버 세션 기반** | 쿠키(SessionId) | Stateful | - 브라우저가 자동으로 서버에 전송<br>- 서버에서 세션 관리<br>- 서버 중심 인증 | - 다른 도메인에서 접근 가능<br>- 세션 탈취 위험<br>- CSRF 공격 가능<br>- 수평 확장 어려움 |
| **Pure JWT Stateless** | 브라우저 저장소 | Stateless | - 서버 상태 제거<br>- SPA 친화적<br>- 프론트에서 토큰 갱신 제어 가능 | - XSS 공격 시 토큰 탈취 가능<br>- 토큰 무효화 어려움 |
| **토큰 회전 모델** | localStorage | 부분 Stateful<br>(Refresh 관리) | - 표준 프로토콜 도입<br>- 장기 로그인 발급 | - Refresh 탈취 시 장기 세션 탈취<br>- 브라우저 토큰 노출 |
| **엣지 세션 모델** ✅ | Secure 쿠키<br>프록시 내부 세션 | 부분 Stateful<br>(Edge 세션) | - BFF 친화적<br>- 브라우저는 세션 쿠키만 보유<br>- 중앙 인증 관리 | - 인프라 복잡성 증가

#### 보안 관점에서의 주요 차이

| 항목 | JWT + HttpOnly 쿠키 (앱 내부 관리) | OIDC 기반 인증 ✅ |
| --- | --- | --- |
| 인증 구조 | 직접 인증/토큰 관리 | 인증을 IdP + 프록시에 위임 |
| 브라우저 저장 | HttpOnly 쿠키에 JWT 저장 | HttpOnly 세션 쿠키만 저장 |
| Access/Refresh 실제 위치 | 애플리케이션 서버 | 프록시 내부 세션 |
| 토큰 검증 주체 | 애플리케이션 서버 | 프록시(oauth2-proxy 등) |
| 애플리케이션의 토큰 인지 여부 | 토큰을 직접 파싱/검증 | 토큰을 몰라도 됨 (헤더 전달 가능) |
| 인가 정책 위치 | 애플리케이션 코드 중심 | IdP + 프록시 + 필요 시 앱 |
| 로그아웃 처리 | 앱이 토큰/세션 무효화 구현 | IdP 세션 + 프록시 세션 중앙 제어 |
| 토큰 회전 책임 | 애플리케이션 | IdP 책임 (프록시는 위임) |
| SSO 확장성 | 앱별 구현 필요 | IdP 기반 중앙 SSO 가능 |
| MSA 적합성 | 중간 (서비스마다 검증 필요) | 높음 (Edge 인증 + 내부 Stateless) |
| 인프라 복잡도 | 낮음 | 높음 |
| Zero Trust 확장성 | 제한적 | 자연스러움 |
| 보안 경계 | 앱 내부에 인증 로직 포함 | **인증 경계가 앱 외부로 분리** |

### ELK 스택 개요

| 구성 요소 | 역할 | 특징 | 주요 사용 사례 | 장점 |
| --- | --- | --- | --- | --- |
| **Elasticsearch** | 분산형 검색·분석 엔진 | - JSON 문서 저장<br>- 고급 검색(Query DSL)<br>- 집계(Aggregation) 기능<br>- 수평 확장 가능 | - 로그 검색<br>- 통계 집계<br>- 이벤트 알람 | - 실시간 분석 가능<br>- 분산 구조로 확장성 높음 |
| **Logstash** | 데이터 수집·필터링·전송 파이프라인 | - Input → Filter → Output<br>- 다양한 플러그인 제공<br>- 데이터 변환/표준화 가능 | - 로그/이벤트 수집<br>- 데이터 전처리<br>- Elasticsearch 전송 | - 다양한 데이터 소스 지원<br>- 복잡한 파이프라인 구축 |
| **Kibana** | 시각화 및 대시보드 | - Elasticsearch 연동<br>- 대화형 쿼리 인터페이스<br>- 차트/그래프/대시보드 제공 | - 로그 분석<br>- 운영 모니터링<br>- 알람 시각화 | - 직관적인 시각화 제공<br>- 문제 원인 탐지 용이 |

### BFF와 CDN 비교

| 항목 | 주 목적 |  주요 기능 | 위치/레이어 |데이터 처리 | 장점 | 단점 |
| --- | --- | --- | --- | --- | --- | --- |
| **Next.js BFF<br>(Backend For Frontend)** | 프론트엔드 서버 사이드 로직 처리 | - API 집계/변환<br>- 서버사이드 렌더링(SSR) | 애플리케이션 서버 레이어 | 동적 요청 처리 | - MSA/SPA와 유연하게 통합 가능 | - 서버 부하 고려 |
| **CDN<br>(Content Delivery Network)** | 콘텐츠 신속 제공 | - 정적 파일 캐싱<br>- 트래픽 분산<br>- 글로벌 지연 최소화 | 인프라/네트워크 레이어 (엣지 서버) | 정적 또는 캐시 가능 | - 오리진 서버 부담 감소<br>- 트래픽 폭주 대응 | - 실시간 동적 데이터 처리 어려움<br>- 캐시 무효화 관리 필요 |
